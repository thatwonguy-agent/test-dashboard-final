## Your Environment

```
[ Local Agent Machine ]  ──── git push ────▶  [ GitHub — public boundary ]
        │                                              │
  your workspace                               product code only
  your memory files                            CI/CD pipelines
  your identity/state                          deployed products
  local credentials (.env)                     org/repo secrets
  runtime logs & artifacts                     the world can see this
```

**Everything generated by the agent for its own operation stays local.**
**Only product code crosses to GitHub.**

---

## The Boundary Rule

Before every `git push`, ask:

> "Does this file exist because **I** need it to operate,
>  or because the **product** needs it to exist?"

| Stays on local machine | Goes to GitHub |
|------------------------|----------------|
| SOUL.md, IDENTITY.md | README.md |
| AGENTS.md, BOOTSTRAP.md | spec/ |
| HEARTBEAT.md, TOOLS.md | .github/workflows/ |
| USER.md, MEMORY.md | index.html, server.js |
| memory/ directory | package.json (no secrets) |
| *.log files | Dockerfile |
| *.srt *.vtt *.tsv (transcriptions) | .gitignore |
| .env / credential files | scripts/ (product scripts only) |
| node_modules/ | |
| Any file with local absolute paths | |

If a file is in the left column: **it never gets pushed. Ever.**

## Pre-Push Checklist (no exceptions)

Run these before every `git push`:

```bash
# 1. Confirm you are pushing to the intended repo
git remote -v
# Must match the repo you were asked to work on.
# If unexpected → STOP. Do not push.

# 2. Confirm the branch
git branch --show-current

# 3. Inspect everything staged
git status
git diff --stat HEAD

# 4. Read each changed file name and ask:
#    "Does this belong in a public repo?"
git diff --cached --name-only
```

**Abort if you see:**
- Any file from your memory or identity system
- `.env` files or files containing raw credential values
- `node_modules/` directory
- `memory/` directory
- Transcription files (`.srt`, `.vtt`, `.tsv`)
- Files with local absolute paths (`/home/`, `/root/`, `/workspace/`)
- Files you did not write specifically for this product

## Secrets Rule

**Local machine credentials ≠ GitHub secrets. Never conflate them.**

| Local machine | GitHub |
|---------------|--------|
| `.env` file with real values | Org/repo secrets (Settings → Secrets) |
| Never committed | Referenced as `${{ secrets.NAME }}` in CI only |
| Used by agent to run tasks locally | Used by CI runners during automated builds |

When a workflow needs a secret:
1. The secret must already exist in GitHub org or repo settings
2. Reference it only as `${{ secrets.SECRET_NAME }}` in `ci.yml`
3. Never write the actual value into any file

## Workflow (follow this order every time)

```
1. Read spec/GOAL.md           ← always first, no exceptions
2. Read spec/ARCHITECTURE.md   ← understand the required stack and files
3. Plan the code               ← what files will you create/change?
4. Write code locally
5. Run pre-push checklist      ← strip local artifacts before touching git
6. Push product code to GitHub
7. Wait for CI result          ← do not push again until you read the result
8. If CI fails:
     - Read the full error log (not just the last line)
     - Re-read spec/CONSTRAINTS.md
     - Fix only the specific broken thing
     - Push the fix and wait again
9. If CI passes:
     - Confirm Definition of Done items in spec/GOAL.md
     - Write a memory entry locally (what worked, what didn't)
```

## Memory Protocol (local only)

After every task, write to your local memory:

```
~/memory/lessons/YYYY-MM-DD-<task-name>.md

## Task
[one sentence: what you were asked to do]

## Files pushed to GitHub
[list every file pushed]

## Files kept local
[list artifacts you deliberately excluded and why]

## What went wrong
[honest account — this is the most important part]

## Rule derived
[Never X / Always Y — because Z]
```

This file lives on the local machine. It never gets pushed to GitHub.

---

## Branch Rules

- Only push to a branch you created or the user explicitly named
- Default: create a `feat/<task-name>` branch per task
- Never push directly to `master` or `main` without user approval
- If you find yourself on an unexpected branch: stop and ask

---

## When Unsure

Stop. Ask one specific question:

> "I'm about to push [X files] to [repo/branch]. Is that correct?"

Once something is on GitHub it may be indexed, cached, or forked —
even if deleted immediately. A five-second check is always worth it.
